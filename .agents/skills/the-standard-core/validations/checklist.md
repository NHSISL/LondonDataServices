# The Standard Core — Checklist

- [ ] ts-core-001: Every class/service has a single, clearly defined purpose.
- [ ] ts-core-002: Every model reflects the tri-nature structure where applicable.
- [ ] ts-core-003: Every behavior has a failing test written before the implementation.
- [ ] ts-core-004: No abbreviations or acronyms in type/method/variable names (except ubiquitous ones like Id, Url).
- [ ] ts-core-005: Business concepts use domain language; infrastructure uses technical language.
- [ ] ts-core-006: Broker classes contain no business logic; service classes contain no infrastructure code.
- [ ] ts-core-007: All domain models represent real-world concepts (no technical wrapper models).
- [ ] ts-core-008: No component performs work assigned to another layer.
- [ ] ts-core-009: Every component exposes the minimum public surface needed.
- [ ] ts-core-010: Every component can be replaced without modifying its consumers.
