# C# Coding Standard — Comments and Documentation — Checklist

- [ ] tsc-csharp-comments-001: Comments are present only where code cannot express intent on its own.
- [ ] tsc-csharp-comments-002: Copyright header uses the exact dashed-line `//` format.
- [ ] tsc-csharp-comments-003: Complex/inaccessible methods are documented with Purposing, Incomes, Outcomes, Side Effects.
- [ ] tsc-csharp-comments-004: No redundant comments that restate what the code already clearly says.
- [ ] tsc-csharp-comments-005: No `/* */` block comment syntax used for copyright.
- [ ] tsc-csharp-comments-006: No XML `<copyright>` tag syntax used for copyright.
