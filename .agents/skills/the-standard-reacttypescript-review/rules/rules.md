# The Standard React + TypeScript + Vite — Review — Rules

## Cite Rule IDs

**tsr-review-001** [ERROR] Every review comment must cite a rule ID from the Standard. A comment without a rule ID is not a Standard-compliant review comment — it is an opinion.

## Separate Architectural and Style Concerns

**tsr-review-002** [ERROR] Architectural concerns (wrong layer, bypassed broker, missing service) must be flagged as `ERROR` and require resolution before merge. Style concerns (naming, formatting) are flagged as `WARNING` or `INFO` and should not block merge unless a specific rule requires it.

## No Personal Preferences

**tsr-review-003** [ERROR] Review comments must not express personal preference (e.g., "I would use X", "this looks cleaner as Y"). Every comment must be grounded in a Standard rule. Personal preference comments must be omitted.

## Identify the Affected Layer

**tsr-review-004** [WARNING] Each review comment must identify the Standard layer affected (e.g., Broker, Foundation Service, View Service, Component, Page). This helps the author understand where the violation sits in the architecture.

## Suggest the Smallest Fix

**tsr-review-005** [WARNING] Suggest the smallest code change that satisfies the violated rule. Do not rewrite entire files or refactor beyond the scope of the violation unless the violation requires it.

## Block Layer Bypasses

**tsr-review-006** [ERROR] Block any PR that bypasses Standard layers — e.g., a component calling a broker directly, a page calling a broker, or a view service calling HTTP directly. These are architectural violations that must not be merged.

## Flag React Doctor Warnings

**tsr-review-007** [WARNING] If React Developer Tools (React Doctor) or equivalent tooling reports warnings about hook usage, excessive re-renders, or missing keys, flag these in the review with the relevant rule ID.

## Reject Non-Compliant Generated Code

**tsr-review-008** [ERROR] AI-generated or scaffolded code must be evaluated against all Standard rules before acceptance. Do not approve generated code that violates any Standard rule simply because it was generated by a tool. Every line must satisfy the same rules as hand-written code.
