# The Standard Team — Core — Checklist

Use this checklist before finalising any purpose document or scenario definition.

---

## Purpose Document

- [ ] Document includes a **Business Nature** section stating what the business is (tst-core-002)
- [ ] Document includes a **Business Goals** section stating what the business aims to deliver (tst-core-003)
- [ ] Document includes a **Future Considerations** section stating anticipated growth (tst-core-004)
- [ ] All three sections are present — none is omitted (tst-core-010)

## Scenarios

- [ ] Every scenario uses exactly three clauses: GIVEN, WHEN, THEN (tst-core-005)
- [ ] Every scenario is written from an end-user's perspective — not an engineer's (tst-core-006)
- [ ] Every THEN clause expresses an outcome a non-technical stakeholder would understand (tst-core-007)
- [ ] No scenario contains API endpoints, HTTP codes, database terms, or UI implementation details (tst-core-009)
- [ ] No scenario omits the actor (GIVEN), the interaction (WHEN), or the outcome (THEN) (tst-core-011)
- [ ] Chained scenarios explicitly reference the prior scenario they build on (tst-core-008)
